# tj-shop v1 architecture research

Background research for the v1 Rust backend. Stack: Axum + SQLx + Postgres +
`sqlx migrate` + Tokio. JSON API only.

**Deployment context** (folded into the recommendations): monorepo with TS
React frontend in `frontend/`, Rust backend in `backend/`. Two deployment
targets are assumed throughout:
- **Local laptop** for development and the TDD inner loop. Postgres runs
  in a Docker container via `compose.yaml`; the Rust backend and frontend
  run natively.
- **A single minimal VPS** for production, fronted by Caddy, with native
  Postgres on the same box and systemd supervising the Rust binary. The
  Postgres install is shared across multiple small apps on the VPS — one
  database + one role per app.

The layout is designed around that pair — no staging, no CI required for
v1, Docker only in dev (never on the VPS). See `recommended-layout-2026-05-15.md` →
"Repo layout (monorepo)", "Deployment topology", "Local iteration ↔ VPS
parity", and "Multiple apps sharing the VPS Postgres".
